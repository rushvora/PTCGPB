Reroll Settings

üë• Friend ID Configuration

üÜî Friend ID
Enter your Pokemon TCG Pocket Friend ID for friend battles and social features. Leave empty if using the ids.txt API instead. This ID will be used across all instances for friend interactions.

‚ö° Instance Management

üì± Instances
Number of regular reroll instances to run. Each instance handles one account simultaneously. Recommended range: 1-50 depending on your system specifications.
‚è±Ô∏è Instance Start Delay
Delay in seconds between launching instances. Helps prevent system overload during startup. Recommended: 2-5 seconds for optimal performance.
üìä Columns
Number of instances per row when arranging windows. Affects the grid layout of your instance windows. Recommended: 3-6 depending on screen size and resolution.
‚≠ê Run Main(s)
Enable Main instance(s) for primary account(s). Main instances can run different logic than reroll instances and are useful for keeping main accounts active while rerolling others.
üéÆ Account Name
Display name for this bot configuration. Used in Discord notifications and logs to help identify different bot setups when running multiple configurations.

‚è∞ Timing Settings

‚ö° Delay
Base delay between actions in milliseconds. Lower values mean faster execution but less stability. Recommended: 200-500ms for optimal balance.
‚è≥ Wait Time
Wait time in seconds for various operations including loading screens and animations. Recommended: 3-10 seconds depending on device performance.
üëÜ Swipe Speed
Speed of swipe animations in milliseconds. Affects pack opening and menu navigation smoothness. Recommended: 200-500ms.
üêå Base Game Compatibility
Enable compatibility mode for slower devices. Increases delays for better stability. Use if experiencing frequent crashes or detection issues.

System Settings

üñ•Ô∏è Display Configuration

üì∫ Monitor Selection
Select which monitor to display instances on. Choose based on your multi-monitor setup. All instances will arrange on the selected monitor according to your column settings.
üîç UI Scale
UI scaling setting for MuMu instances:
  • Scale125: 125% UI scaling (default)
  • Scale100: 100% UI scaling
Must match your MuMu display settings exactly.
üìè Row Gap
Vertical spacing between rows of instances in pixels. Recommended: 50-150 pixels. Adjust based on your screen resolution and preference.

üìÅ File System

üìÇ MuMu Folder Path
Path to MuMu Player installation directory. Usually: C:\Program Files\Netease. Required for launching instances automatically.

üåê Language Settings

üëÅÔ∏è OCR Language
Language for Optical Character Recognition. Must match your game client language for accurate text detection.
Supported: en, zh, es, de, fr, ja, ru, pt, ko, it, tr, pl, nl, sv, ar, uk, id, vi, th, he, cs, no, da, fi, hu, el, zh-TW
üéÆ Client Language
Pokemon TCG Pocket client language. Must match your in-game language setting exactly.
Supported: en, es, fr, de, it, pt, jp, ko, cn

üîß Advanced Options

üöÄ Instance Launch Delay
Delay in seconds when launching MuMu instances. Prevents system overload during startup. Recommended: 3-10 seconds based on system performance.
üìä Auto Launch Monitor
Automatically launch Monitor.ahk with the bot. The monitor script tracks instance health and performance. Recommended to keep enabled.
üéØ Use Role-Based Filters
Enable role-based filtering for advanced users. Provides additional filtering options for specific use cases. Leave unchecked unless you understand its purpose.
üêõ Debug Mode
Enable debug logging and additional information. Creates detailed logs for troubleshooting issues. Only enable when debugging problems.
üî¨ Use Tesseract OCR
Use Tesseract OCR instead of built-in OCR. Requires separate Tesseract installation. May improve text recognition accuracy in some cases.
üí¨ Status Messages
Show status messages during operation. Displays current bot actions and progress. Recommended to keep enabled for monitoring.

Pack Settings

‚≠ê God Pack Configuration

‚ú® Minimum 2‚òÖ Cards
Minimum 2-star cards required to keep a pack. Packs with fewer 2-star cards will be rerolled. Range: 0-5 (0 = keep all packs regardless of stars).

üîÑ Reroll Methods

üìã Method Selection
Choose your reroll strategy:
  • 13 Pack: Traditional method - delete account after opening 13 packs
  • Inject: Advanced account injection system for experienced users
  • Inject Missions: Injection with automatic mission completion
  • Inject for Reroll: Injection with pack count thresholds
1Ô∏è‚É£ 1 Pack Method
Open only 1 pack per account instead of multiple. Enables faster rerolls but provides fewer chances per account.
üí• Menu Delete
Delete accounts through the in-game menu. Only available for non-Inject methods. Provides a cleaner deletion process.
‚è∞ Spend Hourglass
Automatically spend Hourglasses for extra pack openings. Uses in-game currency for additional opportunities. Available for Inject methods.
üéØ Claim Special Missions
Complete special missions during injection. Only available for 'Inject Missions' method. Provides additional rewards and pack opportunities.

üìä Injection Settings

üìà Maximum Pack Count
Maximum pack count for Inject methods. Accounts with more packs will be skipped to focus on newer accounts. Default: 39 packs.
üìâ Minimum Pack Count
Minimum pack count for 'Inject for Reroll' method. Only reroll accounts with fewer packs than this threshold. Default: 35 packs.
üîÑ Sort Order
Sort order for account injection:
  • Oldest First: Process oldest accounts first
  • Newest First: Process newest accounts first
  • Fewest Packs: Process accounts with fewer packs first
  • Most Packs: Process accounts with more packs first

Getting Started Tutorial

üèÅ First Time Setup

Step 1: Basic Configuration
  1. Navigate to Reroll Settings (Ctrl+1)
  2. Enter your Friend ID (optional if using API)
  3. Set Instances to 1 for testing
  4. Configure Account Name for identification
Step 2: System Setup
  1. Go to System Settings (Ctrl+2)
  2. Select your primary monitor
  3. Set Scale to match your MuMu settings (usually Scale125)
  4. Verify MuMu folder path
  5. Set OCR and Client languages to match your game
Step 3: Pack Configuration
  1. Access Pack Settings (Ctrl+3)
  2. Start with "13 Pack" method for beginners
  3. Set minimum 2‚òÖ cards (try 2-3 for testing)
  4. Select 1-2 pack types to focus on
  5. Enable basic card detection if desired
Step 4: First Run
  1. Click Launch All MuMu (F1) to start your instance
  2. Click Arrange Windows (F2) to position properly
  3. Click Start Bot (F3) and confirm settings
  4. Monitor the first few rerolls to ensure proper operation

üìà Scaling Up

Adding More Instances
Once comfortable with single instance operation:
  1. Gradually increase instance count (2, 3, 5, etc.)
  2. Adjust Row Gap for better window spacing
  3. Consider increasing delays if stability issues occur
  4. Monitor system performance and adjust accordingly
Advanced Features
After mastering basics, explore:
  • Inject Methods: More efficient account management
  • Save For Trade: Automatic valuable card detection
  • Discord Integration: Remote monitoring and notifications
  • Multiple Pack Types: Diversified rerolling strategy
